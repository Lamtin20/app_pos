Kê khai thuế (hỗ trợ đối soát)

Hệ thống gom đơn Hoàn thành, nhập kho, chi phí VH (sheet ChiPhiVanHanh), mua TSCĐ trong kỳ. Tỷ lệ thuế ước lấy từ Script Properties (HKD_TAX_VAT_RATE_PCT, HKD_TAX_PIT_RATE_PCT). Hãy chỉnh theo ngành hàng và văn bản hiện hành. File XML là định dạng HkdTaxSupportExport nội bộ, không thay file ký số trên eTax. PDF: mở bản in và lưu bằng trình duyệt (Ctrl+P). Tham khảo: gdt.gov.vn, eTax, Chính phủ.

Chọn quý và bấm «Đối soát kỳ».

API & Google Drive (ảnh sản phẩm, AI)

Các giá trị được lưu vào Script Properties của dự án Apps Script (giống cấu hình máy in). Ô key để trống khi lưu = giữ nguyên key cũ. Sau khi đổi key hoặc quyền Drive, nên Deploy → Phiên bản mới web app và chạy authorizeSunNutMilkNetworkAndDrive() trong Editor một lần nếu được hỏi quyền.

🔐
Bảo mật
Trình duyệt không nhận full key từ máy chủ - chỉ hiển thị dạng rút gọn. Nhập key mới khi cần thay; dùng nút Test để kiểm tra trước khi Lưu.
Gemini (Google AI Studio)

Đang tải…

Hướng dẫn lấy API key Gemini
  1. Mở Google AI Studio → API keys.
  2. Tạo key mới (Create API key), chọn dự án Google Cloud nếu được hỏi.
  3. Dán key vào ô trên → Test → Lưu.
  4. Nếu lỗi quyền mạng: trong Apps Script Editor chạy hàm authorizeSunNutMilkNetworkAndDrive() và chấp nhận quyền.
imgbb - upload ảnh Menu / Mix Hạt
📤
Chọn file ảnh → gửi từ trình duyệt lên api.imgbb.com (không qua Drive). Ô URL nhận link i.ibb.co (thường .png). Lưu key bên dưới trước khi upload.

Đang tải…

Tài liệu & lấy key imgbb
  1. API: api.imgbb.com - POST https://api.imgbb.com/1/upload, tham số key, image (base64), name (tuỳ chọn).
  2. Admin upload bằng FormData từ trình duyệt (tránh IP Google bị chặn).
Nâng cao - proxy HTTPS ngoài Google (hầu như không cần)

Chỉ khi sau này bạn có bất kỳ URL HTTPS nào (không phải Google) nhận POST JSON {key,image,name} và chuyển tiếp form-data sang imgbb. Triển khai chỉ trên Google Sheet thì không cần mục này - bỏ qua.

File mẫu tham khảo trong repo: imgbb-proxy.worker.js (Cloudflare - chỉ mang tính minh hoạ, không bắt buộc).

Google Drive - folder lưu ảnh sản phẩm

Đang tải…

Mở folder đã lưu trên Drive Dùng tài khoản Google của bạn; folder phải được chia sẻ quyền Biên tập cho tài khoản chạy web app (thường là email bạn deploy script).
Hướng dẫn tạo folder & kết nối
  1. Trên Drive, tạo folder (vd. «SUN Ảnh sản phẩm») → mở folder → copy URL từ thanh địa chỉ hoặc chỉ copy ID sau /folders/.
  2. Chia sẻ folder → thêm email tài khoản thực thi web app (Apps Script → Triển khai → «Thực thi dưới dạng») với quyền Biên tập.
  3. Dán ID/link vào ô → Test Drive → Lưu.

Khách hàng

SĐT dùng làm mã định danh. Nếu trùng SĐT thì hệ thống sẽ cập nhật dòng hiện có.

Cách tính Giá Cost trên POS/Admin

1) Giá vốn 1 ly = tổng (định mức nguyên liệu × đơn giá nguyên liệu từ sheet Danh mục/Config) + phụ thu nguyên liệu cỡ ly lớn (nếu khai báo ở «Định mức size») + chi phí vận hành / ly (ống hút, điện nước… nhập tay trong form).

2) Giá bán gợi ý ly lớn có thể nhập tay hoặc suy ra từ biên LN mong muốn (%) trên giá vốn ly lớn.

3) Lưu vào Menu: ghi Giá vốn, Phí VH, giá bán cỡ chuẩn/lớn theo cấu hình Menu (cột phụ thu ly lớn, bật cỡ…).

4) Đơn bán: khi lưu đơn, nếu dòng hàng có costPrice trên JSON món, báo cáo tổng quan / vốn / thuế có thể dùng để ước lượng LN. Nên nhập đủ giá vốn trên Menu.

5) Gói sữa gia đình: phụ thu chai 500ml / 1L là cột riêng trên Menu; cộng vào giá bán như phụ thu cỡ ly.
